# JavaClaw 功能文档

> JavaClaw —— 基于 JavaFX 25 + JDK 21、采用 AgentScope Java 1.0.11 框架的多智能体桌面应用。
> 本文从**功能视角**说明 JavaClaw 能为用户做什么；架构与实现细节见《[设计文档](./agent-design.md)》与仓库根 `CLAUDE.md`。
> 最近更新：2026-06

---

## 1. 产品概述

JavaClaw 是一个面向桌面的多智能体助手：一个**主编排器**统筹一组**领域专家子智能体**，按用户意图自动委派任务、调用工具、自我评估并修正计划。它同时是一个可自学习、可扩展的平台——支持多模型提供商、技能进化、MCP 集成、插件、定时任务与跨工作区隔离。

**三种工作模式**覆盖不同场景：

| 模式 | 一句话定位 | 典型用途 |
|------|-----------|---------|
| 💬 **普通模式** | 主编排器层级委派专家完成工具型任务 | 日常对话、网页操作、收发邮件、系统/桌面自动化 |
| 🧭 **规划模式** | 多专家围桌讨论、互相看到发言、汇总方案 | 方案设计、多角度分析、技术评审 |
| 🛠️ **托管任务模式** | 按 OpenSpec 生命周期把一个工程需求做到验收 | 长时、有明确验收标准的开发/改造任务 |

---

## 2. 多模型提供商支持

内置支持 **5 种模型提供商**，可在设置中切换并分别配置 baseUrl / model / API Key：

| 提供商 | 说明 |
|--------|------|
| **OpenAI 兼容 API** | 任何 OpenAI 协议端点（含本地 LM Studio / vLLM / 第三方网关） |
| **DashScope** | 阿里云通义千问 |
| **Anthropic** | Claude 系列 |
| **Gemini** | Google Gemini |
| **Ollama** | 本地模型 |

- **思考模式**（`model.thinking.enabled`）对支持的模型（DashScope/Anthropic/Gemini 部分模型）开启推理过程展示。
- 所有 Agent 共享同一 HTTP 传输层，避免连接池耗尽。
- **首次启动向导**（OnboardingWizard）三步引导：选提供商模板 → 填 baseUrl/model/Key → 完成，每步可跳过。

## 3. 智能体与专家体系

主编排器根据用户意图，自动把任务委派给最合适的领域专家：

| 专家 | 擅长 |
|------|------|
| 🧑‍💻 编程专家 | 代码、技术、架构问题（纯推理） |
| 📚 知识专家 | 知识问答、概念解释、知识库检索（RAG） |
| 🌐 网页浏览专家 | 网页导航、搜索、表单操作、截图、PDF（Playwright 真实浏览器） |
| 📧 邮件专家 | 邮件收发、搜索、回复（SMTP/IMAP） |
| 🖥️ 系统专家 | 鼠标/键盘/文件/截图等系统操作 |
| 🪟 桌面自动化专家 | 操作任意桌面软件（跨平台，基于截图标注 + Robot） |
| 🔔 通知专家 | 钉钉/企业微信/飞书/邮件/自定义 Webhook 推送 |
| ⌨️ 命令行专家 | 执行命令行 |
| ✅ 任务评估专家 | 多步骤任务完成后的质量评分 |

> 还可在「智能体管理」中**自定义专家**（名称、系统提示词、能力、迭代上限），与内置专家一同参与编排。

**自我优化能力（普通模式）**：编排过程内置目标拆解、执行监控、中段评估与计划修正闭环（GEPA），让智能体在任务过程中自我纠偏，而非一条道走到黑。

## 4. 工具能力

### 4.1 网页自动化（Playwright 真实浏览器）
导航、无障碍树快照与 `@ref` 元素引用、截图标注、点击/输入/键盘/滚动、等待、多 Tab 管理、Cookie、JS 执行、PDF 导出。可结合**站点凭据**自动登录。

### 4.2 系统与桌面自动化
- **系统工具**：鼠标、键盘、文件操作、截图。
- **桌面自动化**：通过截图打标（Set-of-Mark）+ Robot 输入，操作任意桌面软件，跨平台、不依赖 JNA。

### 4.3 邮件
通过 Jakarta Mail 提供 SMTP 发信 / IMAP 收信、搜索、回复。

### 4.4 多渠道通知
钉钉、企业微信、飞书、邮件、自定义 Webhook，可在通知配置中分别接入。

### 4.5 命令行
执行 shell 命令；托管任务内有只读命令白名单（ls/cat/grep 等）免确认直放。

## 5. 技能系统（可自学习）

技能是智能体的「程序性记忆」，记录"遇到某类任务该怎么做"，可由用户编写、也可由智能体**自己沉淀与进化**。

- **技能中心**：浏览/编辑技能（名称、分类、标签、版本），查看使用统计、版本历史并可回滚，管理技能包。
- **三级渐进加载**：常驻只读目录 → 按需拉取单技能全文 → 再拉取单个参考文档，节省上下文 token。
- **自学习闭环**：智能体在对话/任务后自动蒸馏经验，或主动提议新增/修改技能，统一进入**待审提案**队列。
  - `skill.evolution.mode` 三态总闸：`off`（关闭）/ `suggest`（提案待审，默认）/ `auto`（自动落盘 + Toast 提示）。
  - **用户改过的技能受保护**：任何模式下都不会被智能体静默覆盖。
- **JShell 脚本**：技能可携带 `.jsh`/`.java` 脚本，由智能体按需运行（独立 JVM 进程隔离、超时与输出截断，执行前需确认）。

## 6. 知识库与 RAG

- 支持导入文件、文件夹、纯文本构建向量知识库，知识专家在对话中按需检索。
- **全局知识库**（所有工作区共享）与**工作区知识库**（仅当前工作区）两级。
- 切换嵌入模型时需保证 `rag.embedding.dimensions` 与模型实际维度一致，并清空旧向量库。

## 7. 托管任务模式（SDD 工程流程）

把一个工程需求当作一个 OpenSpec change，按生命周期推进到**可验收完成**：

```
提案+评审 → 规格(验收场景) → 设计 → 任务拆解+评审 → 实现(逐项勾选) → 验收+补做 → 归档
```

- **markdown 即真相**：需求/场景/任务清单都是可读的 markdown，进度 = tasks.md 勾选折叠。
- **绝不假完成**：验收由单点权威核验全部能力场景，未过即按未过场景补做；多轮仍不过则升级人工。
- **可暂停/恢复**：中断的任务降为「暂停」等用户手动续跑，不会自动烧 token；续跑从首个未完成项继续。
- **预算闸门**：可设任务级 token 累计预算，达限即停下等用户决定是否加预算续跑。
- **可视化界面**：环形进度 + 统计卡 + 六阶段流水线步进器 + 概览/验收场景/实现清单/实时日志四页签。

## 8. MCP 集成

接入 Model Context Protocol：在设置中配置多个 MCP Server，启动时自动连接、发现其工具并合并进编排器可用工具集，按需转发调用。

## 9. 定时任务

`ScheduleManager` 支持三种触发方式：**间隔**、**每日**、**Cron**。定时任务用独立的执行体运行（不复用交互会话），持久化到工作区 `data/scheduled-tasks.json`。

---

## 10. 站点凭据与会话

按工作区管理站点账号/密码（`SiteCredentialManager`），为 Playwright 浏览器自动登录提供凭据与持久化的登录态（`storageState`），下次直接复用免重复登录。

---

## 11. 工作区隔离

所有配置、数据、日志、浏览器 Cookie、知识库、技能统计、任务索引都**按工作区隔离**。可创建多个工作区分别对应不同项目/账号，切换工作区即整体重载配置与界面。

> 注：技能本体与版本历史是**全局**的（跨工作区共享），但技能使用统计与提案队列按工作区隔离。

---

## 12. 界面与主题

- **9 套内置主题**（2 深 7 浅）：emerald（默认）/ midnight / carbon / sapphire / ocean / plum / terracotta / honey / graphite，运行时实时切换、全局即时生效。
- 切换入口：聊天顶栏「风格」菜单、设置 →「外观 → 界面风格」、通用设置快捷下拉。
- 流式气泡渲染：思考过程与回复在同一气泡内分隔显示；同一工具的连续结果合并、不同工具新建结果块。
- 多媒体附件：支持发送图片、文档、PDF；图片会先转成文字描述供下游处理。

---

## 13. 用量与成本追踪

`TokenTracker` 按会话/日期统计 token 用量，估算人民币成本，并在流式过程中实时更新界面；额外统计**缓存命中率**（命中的 cached tokens 占比），帮助评估上下文复用效率。

---

## 14. 安全与确认机制

高风险工具在执行前会请求确认，分三档：

| 档位 | 表现 |
|------|------|
| 🟢 通知放行 | 仅弹非阻塞 Toast，自动执行 |
| 🟡 弹窗确认 | 需用户点「允许本次 / 永久允许 / 拒绝」（超时默认拒绝） |
| 🔴 二次确认 | 关键操作需输入关键词二次确认 |

- 子智能体发起的确认请求会**透传给用户**，不会被静默跳过。
- 托管任务内有只读命令白名单与轻量范围评估两道免人工通道，但越界操作仍会拦截确认。

---

## 15. 插件系统

支持进程内插件托管（虚拟线程 + 鉴权隔离），目前提供 CHAT 能力接入；插件中心提供界面管理。

---

## 16. 诊断与导出

- **执行轨迹可视化**：`DiagnosticsView` 展示多轮会话的执行轨迹。
- **导出**：`TraceExporter` 可将轨迹导出为 JSON / CSV，便于排查与复盘。

---

## 17. 其他

- **系统托盘**：最小化到托盘后台运行。
- **中文优先**：界面文本、提示词、日志全中文。
- **多会话管理**：侧边栏多会话切换、历史持久化、自动生成会话标题。
